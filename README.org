* Overview

Pollux is a formally verified Protocol Buffers parser/serializer in Rocq. The project proves that parsing inverts serialization using dependent types and the Equations plugin.

*Build:* ~nix develop~ then ~make~

* Admitted Proofs (What's Blocking)

All 5 admitted proofs relate to *4-byte integer parsing* or *varint encoding*:

1. ~Varint.v:94~ - ~encode_valid~: Encoded varints satisfy validity predicate
2. ~Varint.v:99~ - ~encode_correct~: decode(encode(x)) = x
3. ~SimplParse.v:334~ - ~Z32ParseOk~: 4-byte integer roundtrip
4. ~InterParse.v:122~ - ~Z32ParseOk~: Same in intermediate format
5. ~InterParse.v:126~ - ~Z32Length~: Encoded length property

* Architecture Notes

** Parser Framework (parse/)

Uses module functors to abstract over input type:

#+begin_src coq
Module Castor (I : AbstractInput).
  Module Import R := Results(I).
  Module Import P := Parsers(I).
  Module Import S := Serializers(I).
  ...
End Castor.
#+end_src

Key theorem pattern - ~ParseOk~ proves parser inverts serializer:
#+begin_src coq
Definition ParseOk {X} (p : P.Parser X) (s : S.Serializer X wf) :=
  forall x, wf x -> p (s x) = Success x [].
#+end_src

** Format Progression

1. *SimplParse*: Minimal tagged format - validates framework
2. *InterParse*: Ordered fields, defaults, no repeats - top-level correctness proven
3. *ProtoParse*: Full protobuf wire format - field parsing not implemented

** Key Types (Descriptors.v)

Mutually recursive descriptors and values:
- ~ValDesc~: INT, UINT, SINT, FIXED, SFIXED, BOOL, STRING, BYTES, MSG, ENUM
- ~DecoDesc~: IMPLICIT, OPTIONAL, REPEATED
- ~MsgDesc~: Message with reserved IDs and fields

* Dependencies

- Perennial: Verification framework (word/integer helpers)
- Equations: Complex recursive function definitions
- stdpp: Decidable equality, finite maps
- Iris: (via stdpp for gmaps/mapsets)
