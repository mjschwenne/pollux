* Introduction

This guide is designed for Rocq (Coq) users transitioning to Lean 4. It focuses on
syntax mappings, tactic equivalents, and conceptual differences that will help you
leverage your existing proof assistant experience.

** Philosophy Differences

| Aspect      | Rocq                                      | Lean 4                          |
|-------------+-------------------------------------------+---------------------------------|
| Foundation  | CIC (Calculus of Inductive Constructions) | CIC variant with quotient types |
| Proof style | Tactic-focused                            | Term-mode + tactic hybrid       |
| Extraction  | Separate extraction mechanism             | Native compilation to C         |
| Universes   | Set, Prop, Type                           | Prop, Type (unified)            |
| Computation | Limited in proofs                         | Full computation everywhere     |

* Theorems and Proofs

** Stating Theorems

#+begin_src coq
(* Rocq *)
Theorem plus_O_n : forall n : nat, 0 + n = n.
Proof. intros n. reflexivity. Qed.

Lemma plus_comm : forall n m : nat, n + m = m + n.
Proof. intros n m. induction n. ... Qed.
#+end_src

#+begin_src lean
-- Lean 4
theorem plus_O_n : ∀ n : Nat, 0 + n = n := by
  intro n
  rfl

lemma plus_comm : ∀ n m : Nat, n + m = m + n := by
  intro n m
  induction n with
  | zero => simp
  | succ n ih => simp [Nat.succ_add, ih]

-- Or in term mode
theorem plus_O_n' (n : Nat) : 0 + n = n := rfl
#+end_src

** Proof Modes: Tactic vs Term

#+begin_src lean
-- Lean 4: Term-mode proof
theorem and_comm_term {p q : Prop} (h : p ∧ q) : q ∧ p :=
  ⟨h.2, h.1⟩

-- Lean 4: Tactic-mode proof
theorem and_comm_tactic {p q : Prop} (h : p ∧ q) : q ∧ p := by
  constructor
  · exact h.2
  · exact h.1

-- Lean 4: Mixed mode
theorem and_comm_mixed {p q : Prop} (h : p ∧ q) : q ∧ p := by
  exact ⟨h.2, h.1⟩
#+end_src

** Core Tactics

| Rocq              | Lean 4          | Notes                               |
|-------------------+-----------------+-------------------------------------|
| =intros x y z=      | =intro x y z=     | Singular in Lean                    |
| =reflexivity=       | =rfl=             | Term or tactic                      |
| =assumption=        | =assumption=      | Same                                |
| =exact H=           | =exact h=         | Same                                |
| =apply H=           | =apply h=         | Same                                |
| =rewrite H=         | =rw [h]=          | Uses list syntax                    |
| =rewrite <- H=      | =rw [← h]=        | Backwards rewrite                   |
| =simpl=             | =simp=            | Much more powerful in Lean          |
| =unfold f=          | =unfold f=        | Or =simp only [f]=                    |
| =destruct x=        | =cases x=         | Pattern match on hypothesis         |
| =induction x=       | =induction x=     | With structured =with= syntax         |
| =split=             | =constructor=     | Or =And.intro= / =⟨_, _⟩=               |
| =left= / =right=      | =left= / =right=    | Same for disjunctions               |
| =exists t=          | =use t=           | Or =⟨t, _⟩=                           |
| =trivial=           | =trivial=         | Same                                |
| =auto=              | =simp_all=        | Or =aesop=                            |
| =omega=             | =omega=           | Linear arithmetic                   |
| =lia=               | =omega=           | Same solver                         |
| =ring=              | =ring=            | Requires =import Mathlib.Tactic.Ring= |
| =field=             | =field_simp=      | Requires Mathlib                    |
| =discriminate=      | =contradiction=   | Or =simp= often handles this          |
| =injection H=       | =injection h=     | Or pattern match                    |
| =inversion H=       | =cases h=         | With auto-generated equations       |
| =assert (H : P)=    | =have h : P := _= | Or =have h : P := by _=               |
| =pose proof H as X= | =have x := h=     | Introduce new hypothesis            |
| =specialize H x=    | =have h := h x=   | Or =specialize h x=                   |
| =generalize x=      | =generalize x=    | Similar                             |
| =clear H=           | =clear h=         | Same                                |
| =rename H into H'=  | =rename_i h'=     | Slightly different syntax           |
| =exfalso=           | =exfalso=         | Same                                |
| =subst x=           | =subst x=         | Same                                |
| =symmetry=          | =symm=            | Same                                |
| =transitivity y=    | =trans y=         | Or =calc= blocks                      |
| =congruence=        | =congr=           | Similar                             |
| =f_equal=           | =congr_arg=       | As term or tactic                   |

** Induction Tactics

#+begin_src coq
(* Rocq *)
Theorem plus_n_O : forall n, n + 0 = n.
Proof.
  induction n as [| n' IHn'].
  - reflexivity.
  - simpl. rewrite IHn'. reflexivity.
Qed.
#+end_src

#+begin_src lean
-- Lean 4
theorem plus_n_O : ∀ n, n + 0 = n := by
  intro n
  induction n with
  | zero => rfl
  | succ n' ih => simp [Nat.succ_add, ih]

-- Alternative with induction'
theorem plus_n_O' : ∀ n, n + 0 = n := by
  intro n
  induction n
  case zero => rfl
  case succ n' ih => simp [Nat.succ_add, ih]
#+end_src

** Rewriting

#+begin_src coq
(* Rocq *)
Theorem example : forall a b c, a = b -> b = c -> a = c.
Proof.
  intros a b c Hab Hbc.
  rewrite Hab.
  rewrite Hbc.
  reflexivity.
Qed.

(* Multiple rewrites *)
Theorem example2 : forall a b, a = b -> a + a = b + b.
Proof.
  intros a b H.
  rewrite H.
  reflexivity.
Qed.
#+end_src

#+begin_src lean
-- Lean 4
theorem example1 : ∀ a b c, a = b → b = c → a = c := by
  intro a b c hab hbc
  rw [hab, hbc]

-- Or using calc blocks (very powerful in Lean)
theorem example1' (a b c : α) (hab : a = b) (hbc : b = c) : a = c :=
  calc a = b := hab
       _ = c := hbc

theorem example2 : ∀ a b, a = b → a + a = b + b := by
  intro a b h
  rw [h]

-- simp can often handle rewrites automatically
theorem example2' : ∀ a b, a = b → a + a = b + b := by
  intro a b h
  simp [h]
#+end_src

#+begin_src lean
-- Lean 4

  use 0

-- Term mode

theorem exists_elim : (∃ n, n = 0) → True := by
  intro ⟨n, hn⟩

  intro h
  obtain ⟨n, hn⟩ := h
  trivial

* Structural Differences

** Module System

#+begin_src coq
(* Rocq: Module and Module Type *)
Module Type STACK.
  Parameter t : Type -> Type.
  Parameter empty : forall A, t A.
  Parameter push : forall A, A -> t A -> t A.
  Parameter pop : forall A, t A -> option (A * t A).
End STACK.

Module ListStack : STACK.
  Definition t (A : Type) := list A.
  Definition empty {A} : t A := nil.
  Definition push {A} (x : A) (s : t A) : t A := cons x s.
  Definition pop {A} (s : t A) : option (A * t A) :=
    match s with
    | nil => None
    | cons x xs => Some (x, xs)
    end.
End ListStack.
#+end_src

#+begin_src lean
-- Lean 4: Structures/Classes and Namespaces
class Stack (t : Type → Type) where
  empty : t α
  push : α → t α → t α
  pop : t α → Option (α × t α)

instance : Stack List where
  empty := []
  push x s := x :: s
  pop
    | [] => none
    | x :: xs => some (x, xs)

-- Namespaces for organization
namespace MyModule
  def helper : Nat := 42
end MyModule

#check MyModule.helper
#+end_src

** Well-Founded Recursion

#+begin_src coq
(* Rocq: Using Program or Equations *)
From Equations Require Import Equations.

Equations div (n m : nat) (H : m > 0) : nat by wf n lt :=
  div n m H with Nat.ltb n m := {
    | true => 0
    | false => S (div (n - m) m H)
  }.
#+end_src

#+begin_src lean
-- Lean 4: termination_by clause
def div (n m : Nat) (h : m > 0) : Nat :=
  if n < m then 0
  else div (n - m) m h + 1
termination_by n
decreasing_by
  simp_wf
  omega

-- Using well-founded recursion explicitly
def ackermann : Nat → Nat → Nat
  | 0, n => n + 1
  | m + 1, 0 => ackermann m 1
  | m + 1, n + 1 => ackermann m (ackermann (m + 1) n)
termination_by m n => (m, n)
#+end_src

-- Lean 4: ByteArray for binary data
def encodeVarint (n : Nat) : ByteArray := Id.run do
  let mut result := ByteArray.empty
  let mut value := n
  while value >= 128 do
    result := result.push ((value % 128 + 128).toUInt8)

* Common Gotchas

** 1. Termination Checking

Rocq's =Fixpoint= is more lenient; Lean requires explicit termination proofs:

#+begin_src lean
-- This might not pass termination checker without help
def f (n : Nat) : Nat :=
  if n = 0 then 0
  else f (n - 1) + 1
termination_by n  -- explicit termination measure
#+end_src

** 2. Definitional vs Propositional Equality

Lean computes more aggressively; some Rocq =simpl= steps happen automatically:

#+begin_src lean
-- This is definitionally true in Lean
example : 2 + 2 = 4 := rfl

-- But some reductions still need tactics
example (n : Nat) : n + 0 = n := by simp  -- not rfl!
#+end_src

** 3. Coercions

#+begin_src coq
(* Rocq: Explicit coercions *)
Coercion nat_to_Z : nat >-> Z.
#+end_src

#+begin_src lean
-- Lean 4: Instance-based coercions
instance : Coe Nat Int where
  coe := Int.ofNat

-- Or using ↑ notation explicitly
#check (↑5 : Int)  -- 5 coerced from Nat to Int
#+end_src

** 4. Namespaces and Opening

#+begin_src lean
-- Lean uses dot notation extensively
#check List.map        -- full name
#check [1,2,3].map     -- method syntax

-- Opening namespaces
open List in
#check map             -- now just `map` works

-- Scoped opening
open List
#check map
#+end_src

** 5. Syntax Differences Summary

| Concept        | Rocq      | Lean 4           |
|----------------+-----------+------------------|
| Function arrow | =->= or =→=   | =→=                |
| Product type   | =*=         | =×=                |
| Lambda         | =fun x =>=   | =fun x =>=          |
| Forall         | =forall=    | =∀=                |
| Exists         | =exists=    | =∃=                |
| Not equal      | =<>=        | =≠=                |
| And (Prop)     | =/\=        | =∧=                |
| Or (Prop)      | =\/=        | =∨=                |
| Iff            | =<->=       | =↔=                |
| Comment        | =(* ... *)= | =-- ...= / =/-...-/= |
| String         | ="..."=     | ="..."=            |
