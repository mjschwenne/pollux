* Overview

This is a port of the Pollux formally verified protobuf parser/serializer from Rocq to Lean 4.
It targets Lean 4 v4.28.0 with Mathlib v4.28.0 as the only dependency.

The port covers the abstract parser/serializer framework (~Parse~) and the intermediate
format (~InterParse~). The full protobuf layer (~ProtoParse~, ~Varint~, ~SimplParse~, etc.)
has *not* been ported.

All ported files are *sorry-free*; the top-level correctness theorem ~interParseOk~ is proven.

* Building

#+begin_src sh
# Inside the lean/ directory:
lake build
#+end_src

An ~aristotle.nix~ file is present for use with the Aristotle automated theorem prover
(used to fill in proof obligations).

* File Structure

The Lean code mirrors the Rocq ~src/~ layout.

** ~Pollux.Parse~ — Abstract parser/serializer framework

Corresponds to ~src/parse/~ in Rocq.

| Lean file               | Rocq counterpart                           | Notes                                                                   |
|-------------------------+--------------------------------------------+-------------------------------------------------------------------------|
| ~Parse/Input.lean~      | ~parse/Input.v~                            | ~Input~ typeclass + ~ByteInput~ (= ~List UInt8~) instance               |
| ~Parse/Result.lean~     | ~parse/Result.v~                           | ~Level~, ~Data~, ~Result~, ~resultEquiv~                                |
| ~Parse/Parser.lean~     | ~parse/Parser.v~                           | All combinators: bind, concat, or, rep, recursive, etc.                 |
| ~Parse/Serializer.lean~ | ~parse/Serializer.v~                       | Dual serializer combinators with ~wf~ predicates                        |
| ~Parse/Theorems.lean~   | ~parse/Theorems.v~ + ~parse/TheoremsRel.v~ | ~ParseOk~ roundtrip correctness, relational variants, combinator proofs |

Key design difference from Rocq: Rocq uses module functors (~Parsers(InputModule)~);
Lean uses a typeclass (~Input ι~).

** ~Pollux.InterParse~ — Intermediate tagged key-value format

Corresponds to ~src/InterParse/~ in Rocq.

| Lean file                                  | Rocq counterpart               | Notes                                                                                         |
|--------------------------------------------+--------------------------------+-----------------------------------------------------------------------------------------------|
| ~InterParse/Descriptor.lean~               | ~InterParse/Descriptor.v~      | ~Desc~/~Field~, ~Value~/~Val~, sorted-map ops, extensionality, validity, depth/length metrics |
| ~InterParse/Parser.lean~                   | ~InterParse/Parser.v~          | Byte parsers, ~parseValue~ (recursive stateful)                                               |
| ~InterParse/Serializer.lean~               | ~InterParse/Serializer.v~      | Byte serializers, ~serialValue~ (recursive stateful)                                          |
| ~InterParse/Theorems.lean~                 | ~src/InterParse.v~ (§Theorems) | Re-exports the ~Theorems/~ subdirectory                                                       |
| ~InterParse/Theorems/Primitives.lean~      |                                | Primitive roundtrip proofs (byte, unsigned, nat, bool, z32)                                   |
| ~InterParse/Theorems/SortedHelpers.lean~   |                                | Sorted-insert/erase commutativity helpers                                                     |
| ~InterParse/Theorems/Validity.lean~        |                                | ~validDropFirst~, ~validInsert~, depth/encoding-length lemmas                                 |
| ~InterParse/Theorems/SchemaCorrect.lean~   |                                | ~SchemaCorrect~ structural lemmas (~sc_*~)                                                    |
| ~InterParse/Theorems/Compatible.lean~      |                                | ~Compatible~ relation, ~compatibleEqual~                                                      |
| ~InterParse/Theorems/ValList.lean~         |                                | ~valList~ helpers, ~listToValue~ roundtrip                                                    |
| ~InterParse/Theorems/Serialization.lean~   |                                | ~willEncode_nonEmpty~, ~valueWf~/~willEncode~ weakening, ~serialValueInversion~               |
| ~InterParse/Theorems/InterParseOk.lean~    |                                | ~parseOk_wf~ and the top-level ~interParseOk~ theorem                                         |

** Not ported from Rocq

These Rocq files have no Lean counterpart:

- ~Prelude.v~ — not needed (Mathlib replaces Iris/stdpp/Perennial imports)
- ~Varint.v~ — variable-length integer encoding
- ~Descriptors.v~ — full protobuf schema descriptors
- ~ProtoParse.v~ — protobuf wire-format parser
- ~SimplParse.v~, ~InterParse/Tests.v~ — examples/tests
- ~parse/Castor.v~, ~parse/Util.v~ — helpers

* Design Notes

** Sorted-list maps instead of proper map types

~Desc~ and ~Value~ store fields as ~List (Int × _)~ because Lean's kernel positivity
checker rejects mutual inductives that go through ~AList~, ~Finmap~, etc. Map
semantics are recovered via a sorted/no-dup invariant (~WF~) and an extensionality
theorem (~ext_lookup~). The sorted-insert/erase operations and their correctness
proofs are in ~Descriptor.lean~.

** Aristotle integration

Some proofs were generated or completed using the [[https://aristotle.harmonic.fun][Aristotle]] automated theorem
prover. The ~aristotle.nix~ file packages the Python client. Aristotle-generated
proofs tend to use ~grind~, ~aesop~, ~simp_all +decide~, and ~exact?~ heavily.
